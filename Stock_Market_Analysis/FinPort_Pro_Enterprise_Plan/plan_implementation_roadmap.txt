================================================================================
 1. PHASE BREAKDOWN (HIGH-LEVEL)
================================================================================

--------------------------------------------------------------------
PHASE 1: FOUNDATION & ARCHITECTURE SETUP (Weeks 1-4)
--------------------------------------------------------------------
Objective:
  Establish the technical foundation. All repositories bootstrapped.
  Local development environment running (bare-metal processes).
  PostgreSQL schemas created. Kafka cluster operational.

Key Deliverables:
  - 10 Git repos initialized with standard project structure
  - Local setup scripts (shell/powershell)
  - Flyway migrations for all schemas
  - API Gateway routing functional
  - JWT auth flow (register/login/refresh) operational

Exit Criteria:
  - All 10 services boot locally (standard Java -jar execution)
  - PostgreSQL schemas created (all 7 schemas)
  - Kafka cluster running with 10 topics created
  - API Gateway routes requests to User Service
  - JWT login/register flow works end-to-end

--------------------------------------------------------------------
PHASE 2: CORE BACKEND SERVICES (Weeks 5-8)
--------------------------------------------------------------------
Objective:
  Implement the three core CRUD services: Portfolio Service,
  Market Data Service, and Scraper Service. End-to-end data flow
  from scraper to database.

Key Deliverables:
  - Portfolio Service: Full CRUD with auto-fill defaults
  - Market Data Service: Price ingestion + history + caching
  - Scraper Service: NSE + MoneyControl + scheduling
  - End-to-end: Scraper → Kafka → Market Data → DB

Exit Criteria:
  - Create plan + add holdings via REST API
  - Scraper runs on schedule, prices appear in DB
  - Price history queryable via API
  - Redis caching operational for latest prices
  - 80%+ unit test coverage on all three services

--------------------------------------------------------------------
PHASE 3: FRONTEND IMPLEMENTATION (Weeks 9-12)
--------------------------------------------------------------------
Objective:
  Build the Angular application shell, authentication module,
  and the two most critical screens: Plan List and Tactical Dashboard.

Key Deliverables:
  - Angular app shell with enterprise sidenav
  - Login/Register flow (JWT integration)
  - NgRx store (auth, plans, portfolio slices)
  - Plan List screen (CRUD)
  - Tactical Dashboard (KPIs, charts, risk table)
  - WebSocket connection established

Exit Criteria:
  - User can log in and see their plans
  - Tactical Dashboard renders live data from backend
  - KPI cards, charts (ECharts), and risk table functional
  - WebSocket receives price updates
  - Responsive layout on desktop and tablet

--------------------------------------------------------------------
PHASE 4: EVENT-DRIVEN & RISK ENGINE (Weeks 13-16)
--------------------------------------------------------------------
Objective:
  Bring the system to life with real-time event processing.
  Risk Engine recalculates on every price update. Stop-loss
  and target hit alerts flow to the UI.

Key Deliverables:
  - Risk Engine: Full calculation engine operational
  - Notification Service: WebSocket push + email alerts
  - Audit Service: Consuming all events, persisting logs
  - End-to-end: Price change → Risk recalc → UI update

Exit Criteria:
  - Price update triggers risk recalculation within 10 seconds
  - Stop-loss triggers produce a real-time alert in the UI
  - Target hit triggers produce a notification
  - Audit Service logs all events immutably
  - Alert deduplication prevents duplicate notifications
