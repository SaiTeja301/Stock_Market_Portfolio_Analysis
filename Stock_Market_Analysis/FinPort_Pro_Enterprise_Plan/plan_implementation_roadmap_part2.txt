 3. UNIT & TEST STRATEGY
================================================================================

- Unit Testing: JUnit 5 + Mockito (Target: 80% line coverage)
- Integration Testing: Testcontainers for PG/Redis/Kafka
- Contract Testing: Spring Cloud Contract (Events & APIs)
- UI Testing: Cypress/Playwright for E2E critical paths
- Load Testing: k6 (Benchmarking: 50 req/sec, p95 < 200ms)
- Static Analysis: SonarCloud quality gates


================================================================================
 4. CI/CD & DEVOPS PLAN (FINAL)
================================================================================

- Branching: Gitflow (feature -> develop -> main)
- Environments: Dev (Local), Staging (Manual/VM), Prod (K8s/Automated)
- Automation: GitHub Actions + ArgoCD integration in final month
- Registry: GHCR or ECR for versioned Docker images
- Rollback: `git revert` on main triggers auto-rollback via CI/CD


================================================================================
 5. SECURITY & COMPLIANCE
================================================================================

- JWT: RS256 asymmetric keys (periodically rotated)
- RBAC: Role-based Access Control (Admin vs User)
- Rate Limiting: Redis-backed token bucket (100 req/min)
- Secure Scraping: Static proxy rotation + non-identifiable UA strings
- Encryption: TLS 1.3 in transit; AES-256 for DB-at-rest
