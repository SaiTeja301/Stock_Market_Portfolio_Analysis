================================================================================
 1. ARCHITECTURE OVERVIEW
================================================================================

This Angular application is a standalone UI repository that:
- Communicates ONLY via REST APIs and WebSocket to backend microservices
- Has ZERO direct database access
- Uses NgRx for centralized, predictable state management
- All real-time updates flow through WebSocket вҶ’ NgRx Effects вҶ’ Store
- Is fully independent вҖ” can be deployed, scaled, and versioned independently

Repository Name: finport-ui
Tech Stack: Angular 18+ | NgRx | RxJS | ECharts | Angular CDK | Angular Material

================================================================================
 3. ROUTING STRATEGY (Lazy-Loaded)
================================================================================

All feature modules are lazy-loaded. Initial bundle contains ONLY:
- AppComponent (shell)
- CoreModule (guards, interceptors)
- Login route

Route Definitions:
--------------------------------------------------------------------
Path                        Module                Guard          Role
--------------------------------------------------------------------
/login                      AuthModule            None           Public

================================================================================
 5. REAL-TIME INTEGRATION (WebSocket)
================================================================================

WebSocket Service Design:
--------------------------------------------------------------------
- Single persistent WebSocket connection per authenticated session
- Connects to: wss://api.finport.io/ws?token={jwt}
- Auto-reconnect with exponential backoff (1s, 2s, 4s, 8s, max 30s)
- Heartbeat ping every 30 seconds

Incoming Event Types:
  price-updated          вҶ’ MarketActions.priceUpdated()
  portfolio-updated      вҶ’ PortfolioActions.holdingsUpdated()
  risk-recalculated      вҶ’ RiskActions.riskRecalculated()
  stoploss-triggered     вҶ’ RiskActions.stopLossTriggered()
                           + NotificationActions.newAlert()
  target-hit             вҶ’ RiskActions.targetHit()
                           + NotificationActions.newAlert()
  ai-analysis-completed  вҶ’ AiActions.analysisCompleted()

Connection Lifecycle:
  1. User logs in вҶ’ AuthEffects dispatches [WS] Connect
  2. WebSocketService opens connection
  3. NgRx Effects subscribe to incoming messages
  4. On token refresh вҶ’ reconnect with new token
  5. On logout вҶ’ close connection
  6. On disconnect вҶ’ auto-reconnect with backoff

Performance:
  - Messages are debounced at 200ms intervals
  - Batch price updates merged into single state update
  - Only active plan's holdings trigger UI re-render

 8. ROLE-BASED ACCESS CONTROL (RBAC)
================================================================================

JWT Payload:
{
  "sub": "user-uuid",
  "email": "trader@firm.com",
  "role": "TRADER",
  "permissions": ["portfolio.read", "portfolio.write", "holdings.write"],
  "iat": 1709400000,
  "exp": 1709401800
}

Guard Implementation:
--------------------------------------------------------------------
AuthGuard:
  - Checks if JWT exists and is not expired
  - If expired вҶ’ try refresh token
  - If no refresh вҶ’ redirect to /login

RoleGuard:
  - Reads route.data.roles: string[]
  - Compares with JWT role claim
  - If not authorized вҶ’ redirect to /dashboard with toast error

HasRoleDirective:
  - Structural directive: *appHasRole="['ADMIN','TRADER']"
  - Shows/hides DOM element based on current user role
  - Used for buttons, menu items, action columns

================================================================================
 9. ERROR HANDLING & INTERCEPTOR DESIGN
================================================================================

Global Error Interceptor (error.interceptor.ts):
--------------------------------------------------------------------
- Catches ALL HTTP errors globally
- 401 Unauthorized вҶ’ Try token refresh; if fails вҶ’ redirect /login
- 403 Forbidden вҶ’ Show "Access Denied" toast
- 404 Not Found вҶ’ Show "Resource not found" toast
- 422 Validation вҶ’ Extract field errors вҶ’ show inline
- 429 Rate Limit вҶ’ Show "Too many requests, wait..." toast
- 500+ Server Error вҶ’ Show "Server error, try again" toast
- Network Error вҶ’ Show "No connection" banner (sticky)

Error State in NgRx:
  - Each feature state has an error: string | null field
  - Error actions dispatched by effects on API failure
  - Components show error banner via selector

Retry Strategy:
  - 3 retries on 5xx errors with exponential backoff
  - No retry on 4xx errors (user errors)
  - No retry on 401 (auth errors вҖ” handled by refresh flow)

 10. CACHING STRATEGY IN UI
================================================================================

Layer 1 вҖ” NgRx Store (In-Memory State):
  - Plans list:      cached 5 minutes, invalidated on create/delete
  - Holdings:        cached until WS 'portfolio-updated' event
  - Risk metrics:    real-time (always WS-driven)
  - Latest prices:   real-time (always WS-driven)
  - AI analysis:     cached 30 minutes, invalidated on manual refresh

Layer 2 вҖ” HTTP Interceptor Cache:
  - GET /api/prices/:symbol/history вҶ’ cached 1 hour (historical data)
  - GET /api/analytics/performance вҶ’ cached 5 minutes

Layer 3 вҖ” LocalStorage:
  - JWT token + refresh token
  - User preferences (theme, default plan, table columns)
  - Last active planId (for quick resume)

Cache Invalidation:
  - WebSocket events are the primary invalidator
  - Manual refresh button on dashboards
  - Auto-invalidate on navigation if stale > TTL

 13. DESIGN DECISIONS SUMMARY
================================================================================

Decision                             Reasoning
--------------------------------------------------------------------
NgRx over Akita/NGXS                Enterprise standard, Redux devtools,
                                     mature ecosystem, best for large teams

                                     enterprise-proven at scale
OnPush everywhere                    Prevent unnecessary change detection,
                                     mandatory for real-time dashboards
Feature-based folders over           Better encapsulation, independent
type-based (components/services)     deployment of feature areas

                                     like in-app messaging, algo commands)
Dark theme as default                FinTrader industry standard, reduces
                                     eye strain during market hours
JetBrains Mono for numbers           Monospace font ensures price alignment,
                                     industry standard for trading UIs
